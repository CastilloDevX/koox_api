Ko'ox API (Open source)

Nuevo transporte público de Campeche - consulta, administra y explora las paradas y rutas.

Descripción del Proyecto

La API de Ko'ox es una API REST desarrollada con Flask que permite gestionar las paradas de transporte público en Campeche y encontrar la parada más cercana a una ubicación específica. Esta página ofrece una visión general de la API, sus requisitos y cómo utilizar sus endpoints.

Nota: La API es de código abierto y está destinada a fines educativos y de desarrollo. Los datos se almacenan en memoria y no se persisten en el archivo JSON original.

Requisitos

  • Python 3.7 o superior
  • Flask
  • python-dotenv (opcional, para manejar variables de entorno en desarrollo)

Instalación y Ejecución

Para iniciar el proyecto localmente sigue estos pasos:

  1. Clona el repositorio desde GitHub: 
    git clone https://github.com/CastilloDevX/koox_api.git
  2. Activa el entorno virtual y instala las dependencias: 
    venv\Scripts\activate  # Windows
source venv/bin/activate  # Linux/Mac
pip install -r requirements.txt
  3. Ejecuta la aplicación: 
    python app.py

También puedes acceder a la versión desplegada en koox-api.vercel.app.

Endpoints

Devuelve la lista completa de paradas.

GET /stops

Respuesta exitosa:

{
  "ok": true,
  "code_http": 200,
  "body": [
    {
      "Stop_Name": "Parada Central",
      "Latitude": 18.6465,
      "Longitude": -91.8235,
      "Routes": ["Ruta 1", "Ruta 2"]
    }
  ]
}

Devuelve los detalles de una parada específica por su nombre. 

GET /stops/Parada%20Central

Respuesta exitosa:

{
  "ok": true,
  "code_http": 200,
  "body": {
    "Stop_Name": "Parada Central",
    "Latitude": 18.6465,
    "Longitude": -91.8235,
    "Routes": ["Ruta 1"]
  }
}

Crea una nueva parada en el sistema.

POST /stops

Body (JSON):

{
  "Stop_Name": "Nueva Parada",
  "Latitude": 18.6500,
  "Longitude": -91.8300,
  "Routes": ["Ruta 3"]
}

Respuesta exitosa:

{
  "ok": true,
  "code_http": 200,
  "body": {
    "Stop_Name": "Nueva Parada",
    "Latitude": 18.6500,
    "Longitude": -91.8300,
    "Routes": ["Ruta 3"]
  }
}

Actualiza la información de una parada existente.

PUT /stops/Parada%20Central

Body (JSON):

{
  "Latitude": 18.6550,
  "Longitude": -91.8350,
  "Routes": ["Ruta 1", "Ruta 4"]
}

Respuesta exitosa:

{
  "ok": true,
  "code_http": 200,
  "body": {
    "Stop_Name": "Parada Central",
    "Latitude": 18.6550,
    "Longitude": -91.8350,
    "Routes": ["Ruta 1", "Ruta 4"]
  }
}

Elimina una parada del sistema.

DELETE /stops/Parada%20Central

Respuesta exitosa:

{
  "ok": true,
  "code_http": 200,
  "body": {
    "Stop_Name": "Parada Central",
    "Latitude": 18.6465,
    "Longitude": -91.8235,
    "Routes": ["Ruta 1"]
  }
}

Encuentra la parada más cercana a una ubicación geográfica específica utilizando la fórmula Haversine. 

GET /stops/closest?latitude=18.6470&longitude=-91.8240

Respuesta exitosa:

{
  "ok": true,
  "code_http": 200,
  "body": {
    "Stop_Name": "Parada Central",
    "Latitude": 18.6465,
    "Longitude": -91.8235,
    "Routes": ["Ruta 1", "Ruta 2"]
  }
}

Notas Importantes

  • La API busca paradas por nombre sin distinguir mayúsculas y minúsculas.
  • Los datos se almacenan en memoria durante la ejecución del servidor.
  • Las modificaciones no se persisten en el archivo JSON original.
  • La distancia se calcula en kilómetros usando la fórmula de Haversine.
  • El servidor corre en modo debug por defecto; desactiva esta opción en producción.
